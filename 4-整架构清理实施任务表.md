# 4. 整架构清理实施任务表

更新时间：2026-04-18

本文对应以下设计与现状收口目标：

- `newspulse/3-原生AI工作流重构设计.md`
- 当前 `workflow/` 主链路落地后的剩余清理工作

本轮工作的目标不是继续叠加新功能，而是把现有架构中的过渡层、兼容层、旧 AI 外挂层彻底收口，确保后续功能开发只基于原生 workflow 主链路推进。

---

## 1. 清理目标

收口后的目标链路应为：

```text
热榜源 -> 抓取归一化 -> 本地 SQLite
-> Snapshot Stage
-> Selection Stage
-> Insight Stage
-> Localization Stage
-> Render Stage
-> Delivery Stage
```

本次整架构清理需要同时满足：

- Render 不再依赖 legacy adapter，也不再把原生对象中途转回旧 `report_data / standalone_data / ai_analysis`
- Insight、Selection、Localization、Render、Delivery 之间只传递原生 workflow 契约
- `context.py` 不再暴露 legacy 返回值或旧 AI 过渡对象
- `notification/` 只承担发送与消息拆分，不再承载翻译、分析、报告重组
- `newspulse/ai/` 不再承载主业务逻辑，AI runtime 统一收口到 `workflow/shared/ai_runtime`
- 项目真实配置正式切换到 `workflow.selection / workflow.insight / workflow.localization` 阶段导向模型

---

## 2. 完成标准

满足以下条件，视为“整架构已收干净”：

- `workflow/render/legacy.py`、`workflow/selection/legacy.py`、`workflow/insight/legacy.py` 已删除
- `RenderService`、`HTMLRenderAdapter`、`NotificationRenderAdapter` 只消费原生 workflow 报告对象
- `AppContext.run_insight_stage()` 只返回原生 `InsightResult`
- 项目主代码中不再依赖旧 `AIAnalysisResult` 作为主链路中间对象
- `workflow/localization/ai.py` 改为基于 `workflow/shared/ai_runtime`
- `newspulse/ai/` 下旧 analyzer / filter / translator / formatter / client / prompt_loader 不再承担主链路职责，能删则删
- `config/config.yaml` 使用正式 `workflow:` 配置结构，而不是旧 `filter / ai_filter / ai_analysis / ai_translation` 扁平段
- 全量测试通过
- 所有 live AI 测试必须使用真实模型，并人工确认输出质量合格，不能只看测试是否通过

---

## 3. 推荐实施顺序

推荐严格按以下顺序推进：

1. 先清 Render 过渡层
2. 再删除 Insight / Selection 的 legacy adapter
3. 再统一 Localization 到 shared AI runtime
4. 再删除旧 `newspulse/ai/` 业务模块
5. 最后正式切配置模型、统一命名并补收尾回归

原因：

- Render 是当前最大的 legacy 汇合点
- 不先拆 Render，Selection / Insight 的 legacy adapter 很难真正删除
- Localization 如果继续挂在旧 translator 上，AI runtime 仍然是双轨
- 配置层应在链路彻底收口后再正式切，否则会放大迁移成本

建议按以下 commit 粒度推进：

---

## 6. 实施原则

执行本任务表时，遵循以下原则：

- 一次只推进一个可回归、可提交的子任务
- 涉及真实模型的测试必须人工检查输出内容，不接受残缺、空洞、错乱结果
- 删除旧代码前，先确保对应原生链路测试到位
- 不保留长期“双轨”结构，兼容层只允许短期存在于迁移中的单次提交之间
- 所有新开发都必须基于收口后的原生 workflow 架构继续推进

---

## 7. 结束条件

当以下问题都被消灭后，才可以认为“整架构清理”完成：

- 主链路中还有没有任何 legacy adapter
- Render 层还有没有旧格式中转对象
- AI runtime 还有没有双套实现
- 配置文件还有没有旧 flat 配置段
- 包导出还有没有旧兼容符号
- 新功能开发是否还能绕开原生 workflow 主链路直接挂旧模块

如果以上任一项答案仍是“有”，则本任务表还未完成。
